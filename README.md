Змейка (Console Snake)
Классическая игра «Змейка» в консоли Windows, реализованная на C#. Игрок управляет змейкой, собирает еду, растёт и старается не врезаться в стены, препятствия и собственное тело. Игра имеет несколько видов еды с разными эффектами, динамическое ускорение и цветовую графику.

Особенности
Три типа еды:

$ (красный) – обычная: +1 очко, +1 длина.

★ (жёлтый) – золотая: +5 очков, +1 длина.

☠ (тёмно-красный) – гнилая: –1 очко, –1 длина (но не менее 3 сегментов).

Препятствия (#, серый) – непроходимые стены внутри поля. Количество препятствий задано в коде (5 штук).

Динамическая скорость – с каждыми 3 очками задержка между кадрами уменьшается, игра становится быстрее. Минимальная задержка – 20 мс.

Цветная отрисовка: змейка зелёная, еда и препятствия имеют свои цвета.

Границы (решётка #) статичны и не перерисовываются, что исключает мерцание.

Счёт обновляется без лишних перерисовок.

Системные требования
Операционная система: Windows (используются API консоли Windows).

.NET Framework 4.5+ или .NET Core / .NET 5+ (для компиляции и запуска).

Компиляция и запуск
Способ 1: через командную строку (csc)
Убедитесь, что установлен .NET Framework SDK или компилятор C# (обычно входит в состав Visual Studio или .NET SDK).

Откройте командную строку (CMD) или PowerShell в папке с файлом Snake.cs.

Выполните компиляцию:

bash
csc Snake.cs
Запустите игру:

bash
Snake.exe
Способ 2: через .NET CLI (dotnet run)
Установите .NET SDK.

Создайте пустой проект в папке с игрой:

bash
dotnet new console -n SnakeGame
cd SnakeGame
Замените содержимое Program.cs на код игры.

Запустите:

bash
dotnet run
Управление
Стрелки – изменение направления движения змейки.

Esc – выход из игры в любой момент.

Правила игры
Змейка постоянно движется в заданном направлении.

Если голова змейки касается границы, препятствия или собственного тела – игра заканчивается.

Съедание обычной или золотой еды увеличивает длину змейки и количество очков.

Съедание гнилой еды уменьшает длину и счёт.

Чем больше очков, тем быстрее движется змейка.

Возможные доработки
Добавление меню паузы и рестарта.

Система рекордов (сохранение лучшего счёта в файл).

Дополнительные типы препятствий или бонусов.

Звуковые эффекты.

Режимы игры (например, телепортация через границы).
